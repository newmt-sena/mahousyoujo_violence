# 1.作品タイトル・概要
・作品タイトル：『魔法少女☆バイオレンス』  
・ジャンル：選択肢分岐型ノベルゲーム  
・概要：<br>プレイヤーの選択によって物語の結末やキャラクターの好感度が変化する、マルチエンディング形式のノベルゲームです。ゲーム内のテキスト表示や演出（背景・キャラクターの切り替え）、条件分岐システムなど、ノベルゲームとして必要な基本システムを一からすべて自作しました

# 2.開発環境・ツール
・開発言語： C++ (C++20)  
・ライブラリ： OpenSiv3D (v0.6.11)  
・開発環境： Visual Studio 2022  
・データ管理： JSON  

# 3.開発データ
・期間：1ヶ月 / 2026年5月〜6月  
・所要時間：概算で約30時間  
・チーム構成：個人開発  
・担当範囲：企画・シナリオ・プログラム実装のすべて    
JSONデータパース（解析）ロジックの設計・実装  
UIコンポーネント（メッセージボックス、動的サイズ変更ボタン）のクラス設計  
演出コマンド（背景・音響・立ち絵）の自動連続実行システムの構築  
条件分岐、シーン遷移マネジメントの実装  

# 4.外部アセット・フレームワークの利用状況
・使用したもの：  
背景・キャラクターイラスト、効果音・BGM：外部のフリーアセットを使用  
ゲームライブラリ：OpenSiv3D  

・利用状況と詳細：  
ビジュアルおよびサウンド面に関しては、開発期間の短縮とゲーム全体のクオリティ向上のために外部のフリー素材を活用しています。
ゲームシステム（ゲームループやフレームワーク等）に関しては既存のテンプレート等を一切使用していません。
OpenSiv3Dの描画・音声再生などの基本APIのみを用い、ノベルゲームの根幹となる演出制御システムやデータ読み込み処理を一から自作で実装しました。

# 5.技術的なこだわり・工夫した点
・**「データとロジックの分離」による高い拡張性**：  
ゲームのシナリオ進行、演出コマンド（背景・BGM切り替え）、選択肢分岐、パラメーター変化のデータをすべてプログラムから分離し、外部のJSONファイルで一元管理できるように設計しました。これにより、ソースコードを直接書き換えることなく、JSONを編集するだけでシナリオの追加や変更が容易に行える「高い保守性と拡張性」を実現しました。

・**オブジェクト指向に基づくUIのコンポーネント化**：  
メッセージウィンドウ（MessageBox）や選択肢ボタン（Button）を独立したクラスとして設計（カプセル化）しました。ボタンにおいては、文字数に応じた余白（パディング）の動的計算や、マウスホバー時のカーソル形状変更（updateCursorStyle）、カラーパレット（ChoiceColor）による外観の共通管理など、実務を意識して「再利用しやすく、Main 関数の可読性を高めるコード設計」を徹底しました。

・**演出の自動連続実行ロジックの自作**：  
ノベルゲーム特有の「背景変更」や「立ち絵変更」といった演出コマンドを、次のセリフ（テキスト表示）が来るまでループ内で自動的に連続処理するロジックを構築しました。これにより、クリックを挟まずに一瞬で複数の演出（背景を変え、BGMを変え、キャラを出すなど）を同時に切り替える、スムーズなゲーム進行を実現しました。

・**生成AIの主体的な活用と技術の習得**：  
JSONファイルのパース（解析）および保持の仕組みを実装する際、仕組みの深い理解とデバッグ効率化のために生成AI（ChatGPT等）を活用しました。AIのコードをそのままコピー＆ペーストするのではなく、C++におけるデータ構造や安全なデータ取得（contains や isString によるバリデーションチェック）の方法を完全に理解した上で、自作のプロジェクトに最適化して組み込みました。
